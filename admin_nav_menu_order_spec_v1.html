네비게이션 · 메뉴 순서 명세 (전 role)

운영자(6그룹/18라우트)와 외부 4 role(개인 author · CP author · CP master · CP accounting)의 메뉴 순서를 한곳에 정리한다. 외부 role 메뉴는 운영자 메뉴를 권한으로 거른 부분집합이 아니라 role별 독립 메뉴 체계다. 순서는 두 축 — 빈도(자주 쓰는 것 위로)와 권한 그래디언트(민감·고권한 아래로) — 로 정한다.

1

순서 2원칙 — 빈도 ↓ · 권한(민감도) ↑

위로 갈수록 매일·일상·저위험 / 아래로 갈수록 가끔·설정·고권한. 운영자 6그룹의 배열 근거.
▲ 자주 · 저권한빈도↓ 권한↑▼ 가끔 · 고권한
개요
매일 첫 착지. 운영 현황·할 일 큐. 최상단 고정.
콘텐츠
핵심 일상 업무(검수·가격·노출). 가장 자주 들어가는 그룹이라 위쪽.
파트너십
콘텐츠 공급 주체(작가·CP) 관리. 콘텐츠 바로 옆.
커머스
돈·재화. 고위험·저빈도(환불·정산)라 중하단.
운영
주기적 백오피스 도구(알림·배치·감사). 일상 빈도 낮음.
시스템
권한·관리자·초대. 최소 빈도·최고 권한이라 맨 아래.

그룹 내부 순서는 라이프사이클을 따른다 — 콘텐츠: 작품(S-10) → 회차·가격(S-11) → 게시물(S-12) → 모더레이션(S-13) → 노출(S-14). 커머스: 결제(S-30) → 지갑(S-31) → 쿠폰(S-32) → 정산(S-33). 시스템: 관리자(S-50) → 역할(S-51) → 초대(S-52).

개요 1 · S-00

로그인 직후 착지. 매일 보는 화면이라 무조건 최상단. role별 위젯이 갈린다.

콘텐츠 2 · S-10~14

운영자가 가장 자주 쓰는 일상 업무. 라이프사이클 순: 작품 → 회차·가격 → 게시물 → 태그·금칙어 → 큐레이션(노출).

파트너십 3 · S-20~21

콘텐츠를 공급하는 주체. 콘텐츠와 붙어 있어야 자연스럽다. 크리에이터(개인) → 파트너(CP·위임).

커머스 4 · S-30~33

돈·재화. 환불·정산은 고위험·승인 동반이라 일상 업무 아래. 결제 → 지갑 → 쿠폰 → 정산.

운영 5 · S-40~42

주기적 백오피스(알림·배치·감사). 매일은 아님. 감사 로그가 그룹 끝(읽기 전용 기록).

시스템 6 · S-50~52

관리자·권한·초대. 최고 권한·최소 빈도라 맨 아래. 관리자 → 역할(RBAC) → 초대.

동작 규칙 — 접힘 · 활성 · 숨김
