일반 KPI 3개(작품·조회·정산)는 중립/브랜드, 4번째 활성 위임만 Lavender 카드로. 팀장이 발급한 위임이 몇 건 살아있고 몇 건이 만료 임박인지 홈에서 바로 파악. 위임을 안 가진 author(팀이 없는 경우)는 이 카드가 뜨지 않음.

"확인이 필요해요" 카드

흩어진 알림을 홈 최상단에 묶어 행동을 유도. 각 항목은 점 색으로 긴급도(빨강=반려, 초록=확정), 우측에 액션 버튼. 빈 상태일 땐 "지금 처리할 항목이 없습니다"로 안심시킴.

예문 — 검수 반려: "〈비 오는 날의 선택〉 Ep.04 · 사유: 대사 오탈자" [확인] → 회차 수정 화면. 가격 확정: "Ep.03 제안 500PT → 운영 확정 400PT" [보기] → A2.

가장 좁은 메뉴 — 숨김 원칙 (핵심)

CP accounting은 정산·내 정보 2개만. 작가·작품·위임·설정은 회색 비활성이 아니라 아예 렌더하지 않는다. 권한 없는 메뉴를 남기면 "왜 못 누르지?" 혼란만 주기 때문. (이 시안에서는 설명을 위해 "권한 밖(숨김)" 영역에 취소선으로 보여주지만, 실제 화면엔 이 영역이 없다.)

렌더 메뉴 2개숨김 ≠ 비활성
대비 — P1 CP master 6메뉴 ↔ P2 CP accounting 2메뉴. 같은 회사, 같은 디자인, 다른 권한 → 메뉴 구성이 다르다. 이게 "메뉴 자체를 권한으로 분기"의 끝판.
2

정산 — 조회·명세서만
