스펙 시트 — M1

도움말 · 예문 · 카피 · 토큰
1

조회 전용 배지

페이지 제목 옆 상시 노출. 이 화면이 입력 화면이 아니라 조회 화면임을 첫눈에 알린다. 팀원은 데이터를 보기만 한다는 정책의 시각 신호.

색: --info #3A6FE0배경: --info-wash14px / 700 / pill
카피: ● 조회 전용 · 영문 토글 시 Read-only
2

읽기 전용 정책 배너

팀원 권한의 경계를 문장으로 못박는다. "왜 수령 버튼이 없는지"를 화면 안에서 스스로 답하게 하는 핵심 도움말. 닫기 버튼 없음(상시 노출).

확정 카피 — "이 화면은 조회 전용입니다. 정산 수령과 회차 가격 제안은 팀장(작가)을 통해 이루어집니다. 팀원은 본인이 참여한 작품의 통계·정산 예정액·계약 조건을 투명하게 확인할 수 있지만, 직접 입력하거나 수령할 수는 없습니다."
--info-wash + #cdddfa아이콘 원형 i / --info
3

KPI 카드 (투명 조회)

4개 고정. 좌→우 우선순위: 참여 작품 → 참여 회차 → 내 정산 예정액(강조) → 진행 계약. 숫자는 tabular-nums로 자릿수 정렬.

값 28px / 800 / -1px라벨 11.5px / 700 / --ink-3카드 --r 10px / --sh-s
예문: 참여작 0건이면 0 작품 + meta 참여 작품 없음으로 표기(대시 아님).
4

정산 예정액 — 수령 버튼 없음

가장 민감한 카드. 금액은 브랜드 레드로 강조하되, "수령" CTA는 의도적으로 두지 않는다. 대신 Lavender 안내 칩으로 팀장 경유임을 표시 → 위임 체계와 색으로 연결.

값 색: --brand #C7242D안내 칩: --lav-wash / --lav-d우상단 roTag
예문 — 작가(팀장)가 작품 정산을 일괄 수령 → 팀원별 배분율(15%)에 따라 직접 배분. 팀원 화면엔 "예정액"만, "수령됨"은 배분 완료 후 별도 표기.
칩 카피: ◆ 팀장 배분 — 직접 수령 불가
5

참여작 통계 테이블

행 hover 시 #FBFAF9 배경(첫 행에 시연). 클릭은 작품 상세 조회 패널로만 이동 — 편집 진입 없음. 숫자 열 우측 정렬.

헤더 10.5px / 800 / uppercase셀 12.5px / 12px 17px구분선 --line-2
역할 칩 예시: 채색 · 콘티·각색 · 배경 · 글 · 편집
계약 칩 규칙 — RS(수익배분)=파랑, MG(최소보장)=Lavender, 정산완료=초록. 진행/완결 무관하게 정산 종료 행은 정산완료 칩.
6

계약 조건 카드

작품별 내 계약을 카드로 투명 공개. RS면 배분율, MG면 보장액을 노출(조건부 행). 변경 버튼 없음 — 헤더 hint로 "변경은 팀장·운영 통해" 명시.

검수 제출 = 필수 고정 (핵심 규칙)

submit_review는 항상 선택·잠금. 체크 해제 불가(cursor:not-allowed), 필수·고정 뱃지로 명시. 위임의 최소 단위라는 정책을 UI가 강제한다.
